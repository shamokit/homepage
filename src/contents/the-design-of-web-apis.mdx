## ユーザー側に必要ない処理の部分は隠す

- API提供側の立場で作ったAPIはさらす必要のない情報をユーザーに与えてしまうこともあって危険
- ユーザーに不要な情報を与えてしまうとユーザーの手間や混乱が増えてしまう

## REST API
Representational State Transferの略

Representational:再現するみたいな感じ
State:状態とか
Transfer:移すとか伝えるとか

同じAPI（URL）へのリクエストはもう一度リクエストしても同じレスポンスを返す（再現性がある）ような仕組みのあるAPIみたいな感じ？
